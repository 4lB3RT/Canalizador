# Post Plan

Crea un post de planificación en español para X en el hilo de Canalizador.
El tono es **storytelling personal**: cuenta el problema como una historia vivida, presenta la solución como algo que vas a construir, y cierra con un CTA que invite a la interacción.

## Instrucciones

1. Obtén la fecha actual con `date +%Y-%m-%d`
2. Usa el argumento como contexto para definir la funcionalidad
3. Redacta en primera persona, como developer contando qué va a construir y por qué
4. Sigue la estructura y tono del `template.md`
5. Publica con `mcp__x__tweet` usando `reply_to` del `thread.md`

## Argumento

$ARGUMENTS - Describe la funcionalidad que quieres planificar (qué usuario, qué quiere, para qué)
